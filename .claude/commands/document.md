# document

Generate markdown documentation for implemented DriftSentinel features.

## Variables

scope: $ARGUMENTS

## Instructions

1. Review the changed files and the relevant `specs/DS-*.md` documents.
2. Create documentation in `specs/` using filename: `doc-{descriptive-name}.md`.
3. Keep all claims traceable to code, configuration, or observable behavior.
4. Do not write canonical claims into `docs/` when they belong in `specs/`.

## Report

Return only the path to the created documentation file.
