# plan

Create a structured DriftSentinel implementation or design plan.

## Variables

prompt: $ARGUMENTS

## Instructions

- Start with `CLAUDE.md`, `README.md`, and the canonical `specs/DS-*.md` suite.
- If the task needs a new plan file, create it under `specs/` using a concise,
  descriptive filename.
- Keep the plan aligned to the DriftSentinel taxonomy in
  `specs/DS-SDD-001_Architecture_Blueprint.md`.
- Include validation steps and explicit acceptance criteria.

## Report

Return only the path to the created or updated plan file.
